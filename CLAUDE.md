# CLAUDE.md

## 프로젝트 개요

**인천공항 경쟁숙소 가격·잔여 추적기**입니다. 우리 숙소(*Incheon Airport T1 Residence near Paradise City*, 평점 9.4, 부킹닷컴 기준, 총 8실)의 평일 공실을 채우는 것이 핵심 목표이며, 직접경쟁군(굿데이·호텔휴 등) 대비 가격·잔여를 매일 비교해 동적 추천가를 산출합니다.

단일 파일 프로젝트입니다 — `index.html` 하나에 HTML·CSS·순수 JavaScript가 모두 들어 있고, 외부 라이브러리나 빌드 도구는 없습니다.

## 구조

데이터는 `index.html` 안의 `<script>` 블록에 있는 두 JS 객체에 하드코딩되어 있습니다:

- **`DAILY`** — 앞으로 2주(매일 추적). 각 날짜별로 경쟁 4숙소 `{p:가격, r:잔여코드}`, 우리 `{ourN:잔여수, cur:현재가, rec:추천가, ourTrend:[잔여 추이]}`를 담습니다.
- **`WEEKLY`** — D+15~D+30 구간(추세만, 주 1회 갱신).

잔여코드(`r`): `"5"`(5+ 여유) · `"34"`(3~4 중간) · `"lt3"`(<3 임박) · `"so"`(매진).

추천가는 고정값이 아니라 `dynRec()` 함수가 *우리 잔여 × 경쟁사 잔여* 조합으로 동적 계산합니다.

## 매일 갱신 작업

매일 같은 시각에 `index.html`의 `DAILY`(주 1회는 `WEEKLY`도) 객체를 갱신합니다:

1. **우리 잔여** — Sirvoy 실재고 기준(총 8실). 각 날짜의 `our.ourN`과 `ourTrend` 배열에 오늘 잔여를 반영(추이 띠에 점 추가).
2. **경쟁사 데이터** — 부킹닷컴에서 경쟁 4숙소의 가격(`p`)과 잔여코드(`r`)를 확인해 갱신.
3. 상단 기준일 상수 갱신: `ASOF`, `LASTCHECK` (필요 시 `NOTE_GENIUS` 등).

같은 `index.html`을 덮어쓰기로 갱신해 URL을 유지하고, git 커밋으로 매일 이력을 자동 보존합니다.
